## 贡献指南

### 目录结构规范

如果您在某公司有过实习或工作经历，可以按照以下结构创建文件夹和文档。例如，如果您在美团-本地核心商业-美团平台-服体技术部-太平洋技术中心-信息平台组实习过，则应创建如下目录结构：

```
美团/
├── 本地核心商业/
│   ├── 美团平台/
│   │   ├── 服体技术部/
│   │   │   ├── 信息平台组.md      # 信息平台组业务介绍
│   │   │   └── ...
│   │   └── ...
│   └── ...
└── ...
```

如果您发现公司-部门-业务线-小组的业务介绍已经存在，可以在相应的MD文档中继续添加您的业务理解。

### 文档格式要求

当您创建业务介绍文档时，请按照以下格式填写：

```markdown
# 公司-部门-业务线-小组名称

## 基本信息
- **实习类型**：[日常实习/暑期实习]
- **实习时间**：[从xxxx开始，实习了xxx个月]
- **实习岗位**：[具体岗位名称]

## 业务介绍
[详细描述您所在部门/小组的业务内容]


## 工作环境
- **上下班时间**：[具体工作时间]
- **工作地点**：[办公地点]

## 实习生工作内容
[这一部分可以聊一下一般都让实习生干什么工作。如果担心被定位也可以只分享大体方向，以及你感觉对于你的培养力度怎么样]
```

### 示例

以下是一个完整的示例：

```markdown
# 美团平台-服体技术部-信息平台组

## 基本信息
- **实习类型**：日常实习
- **实习时间**：从2024年9月开始，实习了5个月
- **实习岗位**：后端开发实习生

## 业务介绍
美团的客服是统一平台的，而不是每个业务线都有自己的客服。因此，我们组需要为每个业务线的客服接入业务线的信息。比如当你点了一单外卖发现有头发进行投诉的时候，客服在进行判罚的时候，最起码要知道你的个人信息，订单的详细信息，店家的信息等。这一部分信息应该是在到餐业务线中。因此我们需要调用到餐业务线的API，将这些信息传递给客服。

而且我们还负责对业务线接口的统一包装。最简单的例子就是：可能业务线的接口可能会把用户的身份证号返回来。而你需要做脱敏处理。比如只返回前3位和后4位，中间用星号代替。

从整体看，这些需求都比较零碎且繁杂。所以我们内部使用一个接口热部署平台来完成这些需求。就像力扣一样，你只需要写一个函数，然后部署到平台上，就可以被其他业务线调用。

## 工作环境
- **上下班时间**：早上10点上班，晚上8点下班
- **工作地点**：北京关庄鼎城时代广场

## 实习生工作内容
日常实习生基本上就是基于热部署平台来完成日常需求。其实就跟刷简单的算法题一样。不过好处是组内领导和同事人很好，你有大把的时间自学组内热部署服务。组内项目就是你想看就给你权限。这部门的热部署平台是一个很不错的技术，所以如果你能搞明白，上限还是比较高的。
```

## 注意事项

1. 所有内容需遵守相关法律法规，不得包含敏感信息
2. 请客观描述业务情况，避免主观评价
3. 请保持文档格式的统一性

## 提交方式
1.当你有多个部门的业务想要分享的时候，请选择分开提PR
2.git默认不追踪空文件夹，但这个仓库又是一个多层级的仓库。所以请在每个空白层级文件夹下添加.gitkeep文件，用于告诉git这个文件夹是一个空文件夹。
